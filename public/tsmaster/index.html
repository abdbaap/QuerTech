`React.ReactNode` vs. `JSX.Element`

When defining component props or function return types that can accept various renderable types (strings, numbers, elements, fragments), prefer React.ReactNode for its flexibility. JSX.Element is more restrictive and specifically refers to a JSX expression. 

// Prefer React.ReactNode for flexibility
function MyComponent({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>;
}

// JSX.Element is more specific
function SpecificComponent({ element }: { element: JSX.Element }) {
  return <div>{element}</div>;
}

`using` Keyword for Cleaner Test Mocks

The JavaScript using keyword automates the disposal of resources when they go out of scope. In testing, this is invaluable for automatically cleaning up mocks, simplifying test setup and reducing errors from missed cleanup. 

// Example in a testing context (conceptually)
// Assume 'mockServer' has a dispose method
let mockServer;
try {
  mockServer = await createMockServer();
  using mock = mockServer; // Automatically calls mockServer.dispose() when exiting scope

  // Your tests here...
} catch (error) {
  console.error("Test failed:", error);
}
// mockServer.dispose() is automatically called here if it was successfully created.

Type Aliases & `const` Parameters for Strict Typing

Use type aliases to define clear, reusable types, especially for states in finite state machines (FSMs). Employ const type parameters with generics for stricter type inference, preventing unwanted type widening. 

// Type Alias for State
type TState = 'idle' | 'loading' | 'success' | 'error';

// Type Alias to prevent inference widening
type NoInfer<T> = T extends never ? never : T;

// Function using const type parameter for precise inference
function createFSM<
  const TStates extends Record<string, string>,
  TInitialState extends keyof TStates
>(states: TStates, initial: TInitialState) {
  // ... FSM logic
  type State = TStates[keyof TStates];
  type CurrentState = TStates[TInitialState];
  return { states, initialState: initial as CurrentState };
}

// Example Usage
const appStates = { idle: 'idle', loading: 'loading' };
const fsm = createFSM(appStates, 'idle'); // 'idle' is precisely typed
