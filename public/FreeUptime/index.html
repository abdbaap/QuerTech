The Challenge: Aternos Server Shutdowns

Aternos provides fantastic free Minecraft server hosting, but it comes with a catch: servers automatically shut down after a period of inactivity. This tutorial offers a robust, step-by-step technical guide to circumvent this limitation, ensuring your Aternos server remains online 24/7 without recurring costs.

We'll achieve this by deploying an "AFK bot" that simulates a player's presence on your server, utilizing free services from GitHub and Render.com.

Aternos Server Configuration

Before deploying the bot, your Aternos server needs specific settings and plugins to ensure compatibility and smooth operation. Follow these steps carefully within your Aternos panel:

  • Enabled "Cracked" mode: Go to Aternos server options and enable "Cracked." This allows a broader range of Minecraft clients to connect.
  • Selected "Paper" software: Choose "Paper" as your server software. It offers optimal performance and plugin compatibility.
  • Installed "Via" plugins (from modrinth.com): Use Aternos's plugin manager to install these essential plugins:
    • ViaVersion: Essential for enabling newer Java Edition clients (e.g., 1.21.1) to connect to potentially older server versions.
    • ViaBackwards: Allows older Java Edition clients to connect to newer servers.
    • ViaRewind: Facilitates connectivity for 1.8 and 1.7.x clients to newer server versions and vice-versa.
  • Server Restart: After installing plugins, a server restart is crucial for changes to take effect.

GitHub Bot Setup (`Slobos-AFK-BOT`)

We'll use a pre-built AFK bot, `Slobos-AFK-BOT`, and configure it via GitHub:

  • Access GitHub Project: Navigate to the Slobos-AFK-BOT repository on GitHub. Sign in or create an account if you haven't already.
  • Fork the Repository: Click the "Fork" button to create a personal, editable copy of the repository under your GitHub account.
  • Navigate to settings.json: In your forked repository, browse to the `settings.json` file.
  • Edit Server Credentials: Click the "Edit" button (pencil icon). Locate the `server` object and update the `ip` address and `port` number with your Aternos server's details. Crucially, ensure the IP and port values remain enclosed in double quotes.
  • Configure Bot Features: The `settings.json` file allows customization:
    • auto-auth: Useful for cracked servers.
    • anti-afk: Enables bot movement to prevent inactivity kicks.
    • chat messages: Define custom messages the bot sends periodically.
  • Commit Changes: After modifications, commit the changes to your `settings.json` file. Provide a meaningful commit message.

Render.com Deployment

Now, let's deploy your configured AFK bot as a free web service on Render.com:

  • Sign into Render.com: Go to Render.com and sign in or create a free account.
  • Create "New Web Service": From your Render dashboard, initiate the creation of a "New Web Service."
  • Link GitHub Repository: Connect your GitHub account and select your previously forked `Slobos-AFK-BOT` repository.
  • Maintain Default Settings: Keep the default service name, Node.js language, main branch, "Build Command" (`npm install`), and "Start Command" (`node index.js`).
  • Select "Free" Instance Type: This is critical for zero cost. Choose the "Free" instance type (512 MB RAM, 0.1 CPU). Be aware that free instances may spin down with inactivity, causing delays for new requests (though for an AFK bot, this is less impactful).
  • Deploy Web Service: Click "Deploy." Monitor the build logs; initial errors during setup are often normal.

Verification

Confirm your AFK bot is successfully maintaining your server's uptime:

  • In-Game Player List: Once deployed, the bot should automatically join your Minecraft server. Verify its presence by checking the player list in-game.
  • Render.com Status Page: Render.com provides a primary URL for your web service. Visit this URL to see a local page displaying the bot's status (online/running), uptime, current coordinates within the Minecraft world, and the server it's connected to.
